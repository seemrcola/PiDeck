# UI 设计优化落地记录

> 本文记录 `ui-design-audit.md` 中已落地的优化项，以及当前仍需继续推进的事项。审计文档保持为问题和建议清单，本文件作为实现进度记录。

## 已完成

### 主工作台与侧栏

- 左侧项目栏已从偏青蓝背景收敛为中性工作台背景，并通过语义色 token 管理。
- 侧栏行内关闭按钮和主布局左右分隔条已迁移到语义色 token，hover/resize 状态在暗色模式下共用同一规则。
- 侧栏折叠 rail、反馈入口 hover、Agent 状态文字和项目拖拽态已迁移到语义色 token。
- 项目下历史会话默认按项目加载，按更新时间倒序展示，默认 5 条，支持“查看更多”递增加载。
- 历史会话行已改为与 Agent 行一致的紧凑样式，不再显示“历史会话”分组标题、时间、消息数和摘要。
- Agent 与历史会话树行的默认背景、层级连接线、active 连接线和 Header 主按钮阴影已迁移到语义色 token。
- 历史会话支持右键菜单，提供打开、重命名、复制会话、导出 HTML、RPC 日志等会话操作。
- 左侧搜索栏与顶部 logo 区增加间距，底部反馈、配置、设置入口改为靠右对齐。
- 内置 Chat 项目已改用对话图标，不再复用项目文件夹图标；Agent/历史行去掉树形连接线和常驻阴影，搜索框改为统一控件高度，底部入口改为左侧动作组 + 右侧 logo 折叠按钮。
- Header 右侧操作区已按分支上下文、会话动作、面板切换做视觉分组；主操作保留强调，次级按钮降低常驻视觉权重。

### 基础视觉规范

- `styles.css` 顶部已建立字体、字号、行高、语义色、间距、圆角、控件高度、阴影 token。
- 主应用圆角已收敛为小圆角体系，常规容器不再使用大圆角；全屏页面容器不使用圆角。
- Header、Composer、抽屉、右键菜单、设置/配置/反馈入口等高频区域已迁移到 token。
- Composer 的 focus ring、resize handle、shell/silent 状态、发送/停止按钮已迁移到语义色 token。
- 反馈、设置、配置管理已改为全屏页面式弹层，减少窄弹窗带来的空间浪费。
- 自定义窗口标题栏启用时，全屏设置、配置管理和反馈弹层会从标题栏下方开始布局，避免被最大化/最小化控制区遮挡。
- 配置管理已从顶部双 tab 改为全屏左侧导航 + 右侧内容结构，模型、认证、设置、源文件、扩展、技能放入统一导航。
- 工具调用组、工具详情、代码块和 Thinking 区块已收敛为统一小圆角边框卡片，状态色只做轻强调。
- 配置管理深层表单、Skill/Extension 卡片、设置页表单、环境检测、更新弹窗和反馈弹窗进一步迁移到语义 token，减少硬编码色值和旧式大圆角。
- 配置管理侧栏/一级 tab active 状态、错误/诊断卡片、provider 展开态、toast 和新增 provider 输入焦点环已迁移到语义色 token。
- 配置管理表单、SecretInput、模型表格、设置行输入、重命名输入、模型下拉行和测试连接结果已统一使用语义焦点环与状态边框。
- 配置 Raw 编辑器和 Skill 位置选择器的焦点/打开状态已迁移到统一 `--focus-ring`。
- 设置侧栏 active 状态、会话模型 chip、重命名输入和 Web 端点编辑焦点环已迁移到语义色 token。
- 已新增公共 `SelectField` 控件作为基础组件抽象起点，设置页主题、语言和输入快捷键下拉已替换为统一非原生外观。
- 已新增公共 `IconButton` / `CloseIconButton` 控件，设置、配置、反馈、环境检测、RPC 日志和 Codex 导入等弹层关闭按钮已改为图标按钮并补齐可访问标签。
- 固定 `10px` 以上圆角已收敛到 token；图片预览、命令面板、大纲拖拽、设置卡片等旧区域不再单独定义大圆角。
- CSS 中普通固定圆角值已继续收敛为 `--radius-*` token；除 pill、circle 和全屏容器的 `0` 外，普通控件不再直接写固定 `px` 圆角。
- 审计中提到的半像素字号已收敛到字体 token，残留字距样式已改为 `0`，避免 logo、分支控件和终端区出现不一致的文字渲染。
- 散落的 `10px`、`11px`、`12px` 小字号声明，以及工具、设置、Codex 导入等文本区域的 `13px` / `14px` 字号和小徽标行高，已继续迁移到字体 token，减少跨页面重复维护。
- 环境检测状态、侧栏折叠、分支图标、命令/选择器关闭按钮、对话大纲拖拽和配置页图标控件的残留固定字号已迁移到字体 token。

### 暗色模式

- 设置中已增加主题选项：跟随系统、浅色、暗色。
- 应用根节点会根据设置或系统偏好写入 `data-theme`，并通过 CSS 变量切换暗色主题。
- 主工作台、侧栏、Composer、设置页、配置页、反馈页和常见卡片/输入控件已具备暗色基础覆盖。
- 命令面板与模型/思考选择器的边框、hover、选中态、输入框和文本颜色已迁移到语义色 token，避免暗色模式下残留浅色面板细节。
- Header 分支选择器的背景、hover、焦点、计数徽标和空状态已迁移到语义色 token，暗色模式下不再残留白底按钮。
- Header 主按钮 hover/active 阴影和删除确认 danger 边框已迁移到语义色 token。
- 全局 modal、picker 和删除确认遮罩已抽到主题感知 overlay token，反馈输入焦点和 Skill 删除确认说明文字也已改用统一语义 token。
- 工具详情、反馈报告和更新说明中的预格式化代码块已共享 `--color-code-*` token，减少暗色模式下的固定深色块覆盖。
- 右侧抽屉阴影已抽为主题感知 `--shadow-drawer`，调试开关按钮改用统一边框、hover 和 active 语义色。
- 右侧对话大纲浮层的阴影、标题、拖拽柄、计数、展开按钮和列表 hover 状态已迁移到语义色 token，与暗色模式保持一致。
- 主工作台加载遮罩和终端关闭确认的危险按钮已迁移到语义色 token，加载态与确认态在暗色模式下不再使用固定浅色/红色。
- Codex 导入弹窗的工具栏、操作按钮、会话行、状态徽标和导入报告已迁移到语义色 token，减少暗色模式专用覆盖。
- 反馈弹窗底部操作按钮与全局 toast 已迁移到语义色 token，减少浅色主题残留和重复阴影定义。
- 设置页 pi 路径操作按钮、设置行按钮和说明段落已迁移到语义色 token，减少暗色模式下的额外按钮覆盖。
- 历史会话弹窗的副标题、路径条和删除确认 hover 状态已迁移到语义色 token，保持弹窗体系的明暗主题一致性。
- 图片附件预览的清除按钮和消息图片边框已迁移到语义色 token，普通图片控件在暗色模式下不再依赖固定浅色。
- 修改文件列表与回答级文件摘要的边框、文本、hover、成功/运行状态和行数徽标已迁移到语义色 token。
- 环境检测弹窗的成功/错误状态边框和自定义路径输入焦点环已迁移到语义 token，状态卡片在明暗主题下共用同一表达规则。
- 右侧抽屉恢复按钮、会话卡片 hover、重命名按钮 hover 和会话操作 mini-loader 已迁移到语义色 token。
- 主消息流的头像、消息 meta、发送策略徽标、消息气泡和代码复制 hover 状态已迁移到语义色 token。
- 工具调用运行/错误状态的边框、状态点光晕、chip 和 summary 已迁移到语义色 token，输入跳点动画不再使用固定绿色。
- Thinking 流式块、工具/错误消息气泡、用户消息轻强调和空状态文案/Logo 已迁移到语义色 token。
- RPC 日志弹窗的背景、工具栏、筛选按钮、搜索框、日志行、发送/接收方向色和详情面板已迁移到语义 token；终端关闭 hover 与占位状态也不再使用固定红灰色值。
- Composer 外层左右 padding 已收敛为与工作区一致的响应式间距；用户消息操作顺序改为复制、编辑、重发。

### 多语言基础

- 设置中已增加语言选项：跟随系统、简体中文、English。
- 默认语言跟随系统；系统语言非中文时使用英文。
- 已建立轻量 i18n 模块，先覆盖主工作台高频文案。
- i18n 模块已支持变量插值，并新增 `pseudo` 伪翻译语言用于长文案压力测试。
- 中文界面下 `New Session`、`Restart`、`Files`、`Terminal`、`Create Agent`、`Chat`、`Model`、`Think`、`Cycle Model` 等高频英文文案已改为中文。
- Header 停止按钮已接入 `app.stop` 文案，避免中文界面下残留硬编码 `Stop`。
- 配置管理导航、Models/Auth/Settings/Raw/Skills/Extensions 表单的常见按钮、空状态、测试结果、tooltip、确认弹窗已接入中英文文案。
- 反馈弹窗、反馈报告模板、更新弹窗和环境检测弹窗已接入中英文文案。
- 右侧抽屉、历史会话面板、RPC 日志、Codex 导入弹窗、Prompt 建议、模型/思考选择器、Composer 状态、侧栏项目提示、重命名弹层和主工作台低频 toast 已接入中英文文案。
- 配置管理错误边界、配置导入导出、模型拉取、Skill/Extension 操作提示和 User-Agent 预设选项已接入中英文文案。
- 浏览器预览模式的样例会话、历史会话和代理检测提示已改为跟随当前 i18n 文案；预览 settings 现在会保留语言和主题切换结果，便于本地验证暗色模式与伪翻译布局。
- 自定义标题栏的置顶、最小化、最大化/还原和关闭图标按钮已接入中英文 `aria-label` 与 tooltip，避免系统标题栏关闭时残留硬编码文案。
- 设置页 Web 服务端点中的 Host/Port 标签已接入 common 文案，继续减少 JSX 中的固定英文标签。
- 侧栏 Logo 的 `aria-label` 已接入中英文文案，图标级辅助文本不再硬编码英文。
- 环境检测手动路径卡片的命令标签与说明文案已接入中英文资源，避免中文界面残留 `CMD` 标签。
- 配置管理模型列表的 Context/MaxTokens 表头和测试失败详情 Body 标签已接入中英文文案。
- 思考级别选择器的 Off/Minimal/Low/Medium/High/XHigh 档位名称已接入中英文文案。

### 流程与控件

- 环境检测弹窗已增加四步 stepper：检测 CLI、确认路径、验证权限、完成，降低首次启动时的信息密度。
- 终端 Dock 的主题选择已从常驻 select 收进更多菜单，仅保留新建、展开/收起和关闭等高频动作。
- 终端 xterm 运行时字体已改用 PiDeck 等宽字体栈，并从半像素字号收敛到 13px。
- CSS 已补充 PiDeck 品牌字体和展示等宽字体 token；Web 服务 Host/Port 短技术数值改用展示等宽字体。
- `AGENTS.md` 已补充 UI 设计规范、公共组件抽象规则、自定义标题栏适配要求和 PiDeck 字体使用标准。
- 全局交互态已增加统一 `focus-visible` 焦点环，搜索框、配置按钮和源文件选择器等控件补齐键盘焦点反馈；剩余 `transition: all` 已替换为具体属性，并支持系统 `prefers-reduced-motion`。
- 已通过 `npm run typecheck`、`git diff --check`、`npm run build` 以及 renderer-only dev server 启动检查验证当前改动。
- 已通过隐藏 Electron 截图 QA 验证暗色主题、伪翻译文本、设置页横向溢出和全屏弹窗滚动行为。
